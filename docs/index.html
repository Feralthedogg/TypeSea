Overview

Runtime typeguards shaped like a small compiler.

TypeSea treats validation schemas as frontend syntax. Each guard lowers into a cached validation plan — a schema-specialized predicate kernel plus an optimized graph — and executes through the plan, a runtime-compiled validator, or a standalone AOT module.

  • Zero dependencies — no runtime, peer, optional, or bundled packages, enforced by policy.
  • Hostile-input safe — descriptor reads mean user getters never execute.
  • Result-first — expected failures return frozen Result values, never throw.
  • Allocation-aware — successful is() avoids diagnostic allocation.
  • No escape hatches — any, try, and catch are banned from the codebase.

Quick start

Define a guard once, then choose the execution path.

The same guard narrows through is(), returns immutable diagnostics through check(), exports lossless JSON Schema, or compiles into a generated validator. 

npm install typesea
import { compile, t, toJsonSchema, type Infer } from "typesea";

const User = t.strictObject({
  id: t.string.uuid(),
  age: t.number.int().gte(0),
  role: t.union(t.literal("admin"), t.literal("user"))
});

type User = Infer<typeof User>;

// 1) Boolean narrowing — no diagnostic allocation on success
if (User.is(input)) {
  input.id; // narrowed
}

// 2) Immutable diagnostics — frozen Result, never throws
const checked = User.check(input);

// 3) Hot path — generated validator code
const FastUser = compile(User, { name: "isUser" });

// 4) Interop — lossless-only JSON Schema export
const schema = toJsonSchema(User);
⚠

compile() uses new Function, which throws under a Content-Security-Policy that forbids unsafe-eval. In CSP-restricted environments, emit validator source ahead of time with emitAotModule().

⚠

Unsafe FastMode: compile(User, { mode: "unsafe" }) and emitAotModule(User, { mode: "unsafe" }) emit direct property and index reads plus an allocation-free strict-key loop for trusted normalized data. They may execute getters, may accept prototype-backed values, strict objects do not reject symbol or non-enumerable extras, and escaped static property keys may appear directly in unsafe predicate source. mode: "unchecked" also skips strict extra-key loops for already-trusted object shapes. Unsafe and unchecked successful check() results are raw { ok: true, value } objects; failed diagnostics remain frozen and use the same direct-read FastMode object/array/record/discriminant contract where possible.

Semantics

Explicit presence, deliberate edge behavior.

Object presence is a first-class concept, and edge cases that break typical validators are deliberate, documented decisions pinned by tests.

Wrapper Key may be absent Value may be undefined Inferred type
t.optional(inner) yes no key?: T
t.undefinedable(inner) no yes key: T | undefined
t.nullable(inner) — value may be null key: T | null
Input Behavior
NaN, Infinity t.number accepts finite numbers only; t.literal(NaN) matches NaN via Object.is.
Getter-backed properties Never executed — reads go through descriptors, accessors count as missing data.
__proto__, constructor keys Validated as plain own keys through null-prototype lookups; no pollution.
Strict-object extras Rejected via Reflect.ownKeys — including symbol keys and non-enumerable properties.
Sparse array holes Read as undefined without executing accessors.
Global-flag regexes Cloned at construction; lastIndex reset before every test.
UUID strings RFC 9562 versions 1–8 plus the nil UUID.
Cyclic input values Validate finitely via (value × schema) active-pair tracking.
Nesting depth Capped at 1024 frames — deep input fails instead of overflowing the stack.

Architecture

Every schema lowers into a cached validation plan.

The public schema tree is kept for builder validation and diagnostics. Each schema identity lowers once into the Sea-of-Nodes validation IR, is optimized, and is cached as a validation plan owning both a schema-specialized predicate kernel and the optimized graph consumed by generated validators.

Builder Immutable guards validate construction inputs up front.
Frozen schema The tree stores semantics and diagnostic structure.
Sea-of-Nodes IR Lowering shares loads, predicates, keysets, and discriminant dispatch.
Optimize Folding, reachability, and dense node ids keep invariants simple.
Validation plan Cached per schema identity: predicate kernel + optimized graph.

Plan kernel

Drives is() and the check() verdict preflight without per-node interpreter dispatch.

guard.is()

Optimized graph

Source of truth for runtime-compiled and AOT-emitted predicates, and for introspection.

compile() · emitAotModule() · graph()

Diagnostic collector

Failed check() calls replay the schema-aware collector for issue paths and codes.

guard.check()
🔒

Injection-safe codegen: user-controlled literals, regexps, object keys, keysets, and dynamic schema fallbacks live in side tables referenced by numeric index. Hostile property names cannot escape into generated source — pinned by dedicated injection-audit tests.

Ecosystem adapters

Structural adapters without importing frameworks.

Adapter functions return objects or functions shaped for the target ecosystem. TypeSea does not import tRPC, Fastify, or React Hook Form at runtime.

Target Export Behavior
tRPC toTrpcParser / toAsyncTrpcParser Parser objects that return decoded values or throw TypeSeaAssertionError.
Fastify toFastifyRouteSchema Lossless JSON Schema fragments for body, querystring, params, or headers.
Fastify toFastifyValidatorCompiler Route validators dispatched by httpPart, producing { value } or { error }.
React Hook Form toReactHookFormResolver Async resolver returning a nested field-error tree that RHF traverses by path.

Release gate

Publishing requires the full local gate.

One script runs everything CI runs: source policy, docs smoke, typecheck, lint, tests, build, dist policy, public API snapshot, package contents, clean consumer install, benchmark smoke, and pack dry run. CI executes the same gate on Node 20.19, 22, and 24; releases publish with npm provenance. 

npm run release:check
🛡

prepack re-runs the full check, so even a bare npm publish cannot skip the gate. Package policy rejects any runtime dependency fields before the tarball is accepted.
